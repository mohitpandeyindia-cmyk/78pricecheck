# SEVENTYEIGHTOS: Coding & Development Rules

This document outlines the mandatory coding rules and constraints for development on the **SEVENTYEIGHTOS** (78OS) project and the **78 PriceCheck** application. All developers and agents must adhere strictly to these guidelines.

---

## Core Development Principles

1.  **Architecture Integrity**: Never redesign the approved architecture.
2.  **Schema Consistency**: Never change the database schema without approval.
3.  **Interface Stability**: Never change API contracts without approval.
4.  **Milestone-Driven Progress**: Complete one milestone before starting the next.
5.  **Sequential Review Gates**: Stop after every milestone and wait for review.
6.  **Scope Discipline**: Do not implement features outside the Version 1 scope.
7.  **High-Performance Lookups**: Keep barcode lookup optimized for under one second.
8.  **Strict Language Standards**: Write clean, production-quality TypeScript.
9.  **Dependency Minimization**: Avoid unnecessary dependencies.
10. **Modular Architecture**: Maintain clear separation between frontend and backend.
11. **Speed Goal alignment**: Every change should support the 3-second customer experience goal (loading, scanning, and displaying results within 3 seconds).
